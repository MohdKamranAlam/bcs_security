# BCS-256 Curve Specification

## Status

BCS-256 is a Qur'an-inspired, rigidly generated, verified 256-bit prime-order elliptic-curve research candidate for classical ECC, secure only with strict point validation and not post-quantum without ML-KEM hybridization.

This document freezes the current BCS-256 parameters for independent verification, implementation testing, and external review. It is not a standards claim.

## Curve

The curve is defined over the prime field `F_p`:

```text
E: y^2 = x^3 - 2x^2 + 5x + 4 mod p
```

```text

```

## Security requirements

- **Point validation:** Every received public point must be checked to satisfy the curve equation before scalar multiplication.
- **Scalar range:** Private scalar must be in `[1, n-1]`.
- **No secret logs:** Private scalars and shared secrets must never be logged.
- **KDF:** Raw ECDH output must be passed through HKDF-SHA-256 or HKDF-SHA-512.
- **AEAD:** Encryption must use AES-256-GCM or ChaCha20-Poly1305.
- **PQ mode:** For post-quantum security, combine BCS ECDH with ML-KEM and derive the final key with HKDF over both shared secrets.

## Validation algorithm

Given encoded point `(x, y)`:

```text
1. Reject infinity as a public key.
2. Reject if x >= p or y >= p.
3. Reject unless y^2 mod p == x^3 - 2x^2 + 5x + 4 mod p.
4. Because #E(F_p)=n is prime and h=1, no extra subgroup check is required for valid curve points.
```

## ECDH protocol

```text
private_key d in [1, n-1]
public_key Q = dG
peer public key R must pass strict validation
shared point S = dR
raw secret = I2OSP(S.x, 32)
final key = HKDF(raw secret, salt, info)
```

If `S` is infinity, reject.

## Known audit state

- `p` and `n` pass strong BPSW primality checks in Python/SymPy.
- Main-curve ECDLP cost is approximately `2^128` group operations by Pollard rho.
- MOV/pairing attack lower bound is strong in the current audit.
- Curve is not anomalous (`p != n`).
- Twist security remains open until the large composite twist factor is fully factored.
- Strict point validation is mandatory and is the formal mitigation for twist attacks.

## Required final-grade proof artifacts

- Sage/Magma primality certificates for `p` and `n`.
- Independent Sage/Magma proof that `E.cardinality() = n`.
- Exact embedding degree `k = ord_n(p)`.
- Full twist factorization or formal validation-only security policy.
- Repeated audit for BCS-521 after the 521-bit candidate is finalized.
