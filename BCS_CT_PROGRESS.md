## ⏭ Next session — Milestone 2 (Point arithmetic + ladder)

### Step 1 — Change of variables

Our curve is `y² = x³ − 2x² + 5x + 4`.  The non-zero `a₂ = −2` term means
the standard short-Weierstrass formulas can't be applied as-is.  We
substitute `x → x + 2/3` (mod p) to land in short Weierstrass form
`y² = x³ + A·x + B` for some `A, B ∈ F_p`.

Concretely:

```text
x  =  x' + 2/3
y² =  (x' + 2/3)³ − 2·(x' + 2/3)² + 5·(x' + 2/3) + 4
   =  x'³  +  A·x'  +  B
```

where `A = 5 − 4/3` and `B = 4 + 10/3 − 8/27 − 4/3 · 2/3` (expand
carefully; we'll have the Python script emit the final A, B in
Montgomery form).

The generator `G = (0, 2)` in the original chart becomes
`G' = (−2/3, 2)` in the short-Weierstrass chart.  Public-key bytes
remain in the **original** chart for backward compatibility; the
internal point ops use the short-W chart.

### Step 2 — `src/ct/point.rs`

```rust
pub struct ProjPoint {
    pub(crate) x: Fp521,   // Montgomery form
    pub(crate) y: Fp521,
    pub(crate) z: Fp521,
}
// Z = 0  ⇒  point-at-infinity O.
```

### Step 3 — Renes-Costello-Batina complete add + doubling formulas

From EUROCRYPT 2016 §3 (Algorithm 1 & 2 with `a ≠ 0`).  Both formulas
are branchless and handle `P + O`, `O + Q`, `P + P`, `P + (-P)`
uniformly.  Cost:  **add = 12M + 0S + 3·b·M + 23A**,
**double = 8M + 3S + 7·b·M + 19A**.

### Step 4 — `src/ct/ladder.rs`

```rust
pub fn scalar_mul_ct(k: &Scalar, p: &ProjPoint) -> ProjPoint {
    let mut r0 = ProjPoint::identity();
    let mut r1 = p.clone();
    for bit in k.bits_msb_first() {                  // exactly 521 iter
        let b = Choice::from(bit);
        ProjPoint::conditional_swap(&mut r0, &mut r1, b);
        r1 = r0.add(&r1);
        r0 = r0.double();
        ProjPoint::conditional_swap(&mut r0, &mut r1, b);
    }
    r0
}
```
